# 最优记忆系统架构 v2.0

**实施日期：** 2026-02-28  
**目标：** 解决跨session记忆不一致问题，优化token消耗

---

## 系统架构

### 设计原则

1. **分层存储** - 类似人脑：短期记忆 → 中期记忆 → 长期记忆
2. **Token优化** - 每层精炼摘要，避免上下文超限
3. **快速检索** - 关键词索引，秒级定位历史信息
4. **自动维护** - 心跳任务定期整理，无需人工干预

---

## 核心组件说明

### 1. GLOBAL_CONTEXT.md
**类型：** 实时状态跟踪  
**更新频率：** 每次重要对话后  
**内容：**
- 当前项目状态
- 待办事项（等待确认/自动执行）
- 最近的决策/事件时间线
- 团队架构
- 重要文件索引

### 2. KEYWORD_INDEX.md
**类型：** 快速检索索引  
**更新频率：** 实时  
**内容：**
- 项目关键词映射
- 技术关键词映射
- 决策记录
- 待办跟踪

**使用方法：**
```
用户问："之前那个迪荡湖项目怎么样了？"
→ 查KEYWORD_INDEX → 找到"迪荡湖"关键词
→ 关联到 GLOBAL_CONTEXT.md 中的项目状态
→ 给出准确回答
```

### 3. 自动摘要系统
**脚本：** `scripts/generate_session_summary.sh`  
**触发：** 每次会话结束时  
**输出：** `memory/sessions/session-{ID}-{时间}.md`

**摘要内容：**
- 关键决策点
- 待办事项
- 重要信息
- 参考文件

### 4. 启动自检清单
**文件：** `STARTUP_CHECKLIST.md`  
**执行：** 每次启动时自动遵循  
**作用：** 确保不遗漏任何一层记忆

## 工作流程

### 启动流程
```
1. 读取 SOUL.md + USER.md + IDENTITY.md
   → 建立身份认知
   
2. 读取 GLOBAL_CONTEXT.md + KEYWORD_INDEX.md
   → 获取全局状态
   
3. 读取 memory/今日.md + memory/昨日.md
   → 加载近期记忆
   
4. （按需）读取 MEMORY.md 或工作流手册
   → 加载长期知识
   
5. 更新 GLOBAL_CONTEXT.md 中的"最后启动时间"
```

### 对话过程中
```
重要决策/事件
    ↓
实时更新 GLOBAL_CONTEXT.md
    ↓
更新 KEYWORD_INDEX.md（如需要）
    ↓
（可选）生成本次会话摘要
```

### 心跳维护（每4-6小时）
```
1. 检查待办事项状态
2. 整理今日记忆到归档
3. 更新 MEMORY.md（如需）
4. 清理过期会话摘要（>7天）
```

## 错误处理

### 如果不记得某些信息

| 场景 | 错误做法 | 正确做法 |
|------|----------|----------|
| 不记得项目名称 | "我不记得了" | 查 GLOBAL_CONTEXT.md → "当前项目：迪荡湖酒店+住宅" |
| 不记得待办事项 | "我不知道" | 查 GLOBAL_CONTEXT.md 待办列表 → "等待确认进入概念设计阶段" |
| 不记得某个关键词 | "没听说过" | 查 KEYWORD_INDEX.md → 找到关联信息 |
| 不记得很久以前的事 | "不记得了" | 查 MEMORY.md → 或搜索 memory/sessions/ |

## Token消耗对比

| 方案 | 启动Token | 优点 | 缺点 |
|------|-----------|------|------|
| **加载所有原始会话** | 50K+ | 完整历史 | ❌ 超限、慢、噪声多 |
| **只加载当前session** | 1K | 快速 | ❌ 不记得之前的事 |
| **分层记忆（本方案）** | 4.5K | ✅ 平衡完整性和效率 | ✅ 跨session一致 |
