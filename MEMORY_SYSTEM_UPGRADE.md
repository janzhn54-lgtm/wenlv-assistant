# MEMORY_SYSTEM_UPGRADE.md - 记忆系统升级方案

## 问题诊断

**现状问题：**
- 每个session独立，只加载自己的历史
- 文件记忆（memory/日期.md）需要主动读写，容易遗漏
- 跨session对话时，出现"不记得"的尴尬

**用户期望：**
- 作为"小白"，应该记得所有对话
- 不需要重复解释之前说过的事情
- 一致的上下文体验

---

## 解决方案

### 方案1: 会话聚合器（推荐）

**机制：**
每次启动时，自动检查并汇总最近24小时的所有会话摘要

**实现：**
```bash
# 启动时执行
1. 读取 memory/YYYY-MM-DD.md (今日记忆)
2. 读取 memory/YYYY-MM-DD.md (昨日记忆)
3. 检查其他活跃session的最近消息
4. 生成统一的上下文摘要
```

**优点：**
- 自动聚合，无需手动干预
- 轻量级，只汇总摘要而非全文

---

### 方案2: 全局记忆文件

**机制：**
建立一个 `GLOBAL_CONTEXT.md`，实时记录关键信息

**结构：**
```markdown
# 全局上下文

## 当前项目状态
- 项目名称：迪荡湖酒店+住宅
- 当前阶段：调研分析完成，等待确认
- 下一步：概念设计

## 最近的决策
- 2026-02-28 03:38: 启动大规模学习模式
- 2026-02-28 00:58: 确认10人团队架构

## 待办事项
- [ ] 爸爸确认进入概念设计阶段
- [ ] 安装软件工具链

## 重要参考文件
- STUDIO_WORKFLOW.md - 完整工作流手册
- 小白设计工作室_完整工作流手册_V1.0.md
```

**更新机制：**
- 每次重要对话后自动更新
- 心跳任务定期整理

---

### 方案3: 启动自检清单

每次启动时，执行以下检查：

```markdown
## 启动自检清单

- [ ] 读取 SOUL.md - 确认身份
- [ ] 读取 USER.md - 确认用户信息
- [ ] 读取 MEMORY.md - 长期记忆
- [ ] 读取 GLOBAL_CONTEXT.md - 全局上下文
- [ ] 读取 今日+昨日 memory 文件
- [ ] 检查其他活跃session摘要
- [ ] 确认当前项目状态
```
